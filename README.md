# USV Guardian Hub

Платформа управления автономными надводными аппаратами (USV — Unmanned Surface Vehicle) для мониторинга качества воды на **Капшагайском водохранилище** (р. Или, Алматинская область, Казахстан).

Веб-приложение позволяет:

- видеть положение парка USV в реальном времени на интерактивной карте водохранилища,
- управлять миссиями (планировать маршруты, запускать, аварийно останавливать, возвращать на базу),
- просматривать измерения качества воды (pH, растворённый кислород, мутность, температура, глубина, индекс загрязнения) с привязкой к координатам,
- настраивать пороги нормы по каждой метрике и моментально видеть, как меняется оценка качества,
- выгружать сводные отчёты в PDF и CSV за выбранный период и устройство.

## Возможности

### 🗺️ Оперативная карта
- Стилизованная карта Капшагайского водохранилища — реальная вытянутая W→E форма с устьем р. Или и притоком р. Чилик.
- Многослойная вода (радиальный градиент, ряби, дрейфующий шиммер), светящаяся береговая линия, батиметрические контуры.
- Маркеры роботов: индикатор курса, гало-пульсация цвета аппарата, статус-пип (online / mission / RTL / offline).
- Точки проб: интерактивные, при наведении показывают мини-карточку со всеми извлечёнными данными и координатами; кластеризация при отдалённом виде.
- Pan / zoom (Wheel + drag), компас-розетка, шкала (5 / 10 км), легенда, переключатели «Следы движения» и «Сетка».

### 🤖 Управление аппаратами
- Карточка робота с телеметрией: батарея (sparkline за 30 с), сигнал, скорость, курс, GPS, прогресс по точкам маршрута.
- Редактор маршрута прямо на карте — клики добавляют waypoint'ы, можно удалять, очищать, запускать миссию.
- **АВАРИЙНАЯ ОСТАНОВКА** и **ВОЗВРАТ НА БАЗУ (RTL)** через `AlertDialog` с подтверждением; событие пишется в журнал робота с критическим уровнем серьёзности.
- Журнал событий по каждому роботу с фильтром по типу (подключение, потеря связи, низкий заряд, RTL, ESTOP, старт/конец миссии).

### 📊 Аналитика и отчёты
- Фильтры по периоду (с / по) и устройству; шапка показывает количество измерений и средний индекс качества.
- Графики динамики pH, O₂, мутности и температуры (recharts).
- **CSV-экспорт** — реальный файл с UTF-8 BOM (корректно открывается в Excel с кириллицей), 15 колонок включая координаты и метки качества.
- **PDF-экспорт** — формируется HTML-отчёт (A4 landscape) и открывается с автоматическим запуском диалога печати; пользователь выбирает «Сохранить как PDF» в браузере. Если pop-up заблокирован — отчёт скачивается как `.html` (fallback).

### ⚙️ Пороги качества воды
- Управление границами нормы и критики для **pH** (4 порога), **O₂**, **мутности**, **температуры** и **индекса загрязнения**.
- Предпросмотр пересчёта в реальном времени: средний индекс качества и распределение проб по тонам (норма / удовл. / плохо / критическое) пересчитываются на лету ещё до сохранения.
- Сохранение в `localStorage` (`usv.thresholds.v1`); кнопка сброса к стандартным значениям.
- Изменённые пороги автоматически применяются в карточке пробы, в индикаторах на карте и в выгружаемых отчётах.

## Запуск

Нужен установленный [Bun](https://bun.sh/) (≥ 1.0).

```bash
# установка зависимостей
bun install

# dev-сервер (Vite + HMR)
bun run dev

# production-сборка
bun run build

# превью production-сборки
bun run preview

# линтер
bun run lint

# форматирование
bun run format
```

После запуска `bun run dev` приложение доступно на `http://localhost:3000` (или порт, указанный Vite).

### Деплой на Cloudflare Workers

Проект настроен под Cloudflare Workers через [wrangler](https://developers.cloudflare.com/workers/wrangler/):

```bash
bun run build
bunx wrangler deploy
```

Конфигурация — в [wrangler.jsonc](wrangler.jsonc).

## Метрики качества воды

Приложение оценивает интегральный **индекс качества (0–100)** на основе шести измерений. Все пороги настраиваются в разделе «Пороги качества» — изменения мгновенно применяются по всему UI и в отчётах.

| Метрика | Единицы | По умолчанию: норма | Что делает порог |
|---|---|---|---|
| **pH** | — | мягкие 6.8–8.2, жёсткие 6.5–8.5 | вне мягких = −10 баллов, вне жёстких = −25 |
| **Растворённый кислород (O₂)** | мг/л | ≥ 6 норма, < 4 критично | ниже нормы = −12, ниже критики = −30 |
| **Мутность** | NTU | ≤ 5 норма, > 8 критично | выше нормы = −10, выше критики = −25 |
| **Температура** | °C | ≤ 24 | выше = −10 |
| **Индекс загрязнения** | 0–100 | < 20 низкий, < 45 умер., < 70 высокий, ≥ 70 крит. | штраф = (значение − 20) × 0.5 |

Итоговая оценка:

- **80–100** — Отлично
- **60–79** — Хорошо
- **40–59** — Удовлетворительно
- **20–39** — Плохо
- **0–19** — Критическое

Логика собрана в одной функции [`assessQuality(sample, thresholds)`](src/components/app/thresholds.ts) и переиспользуется на карте, в карточке пробы и в отчётах.

## События и журнал

`useEventLog` отслеживает переходы статусов роботов и записывает события:

| Тип | Когда возникает | Серьёзность |
|---|---|---|
| `connected` | offline → online/mission | success |
| `disconnected` | * → offline | danger |
| `low_battery` | заряд пересёк 20 % вниз | warning |
| `rtl` | переход в RTL (ручной или автоматический) | critical |
| `estop` | подтверждённая аварийная остановка оператором | critical |
| `mission_start` | старт миссии | success |
| `mission_done` | завершение миссии | success |

Ручные команды ESTOP и RTL дополнительно пишут в журнал источник: `«оператор · ручное подтверждение»`.

## Симуляция

Реальное оборудование пока не подключено — все данные генерируются:

- `useRealtimeSimulation` обновляет позицию каждого робота каждые ~1.5 с, продвигая по waypoint'ам, обновляя курс, скорость, батарею и сигнал, и складывая trail.
- `initialSamples` в [mock-data.ts](src/components/app/mock-data.ts) — 32 пробы, распределённые вдоль центральной линии озера через `pointOnLake(t, lateral)`.
- Координаты в проценте экрана конвертируются в реальные градусы вокруг центра водохранилища (43.88° N, 77.07° E).

Чтобы подключить настоящее железо, нужно заменить `useRealtimeSimulation` на WebSocket / MQTT клиент, который будет push'ить телеметрию в то же состояние `robots`.
