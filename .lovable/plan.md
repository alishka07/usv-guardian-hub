# План доработки USV-платформы

## 1. Имитация реального времени (тикер)
- Добавить хук `useRealtimeSimulation` в `src/components/app/`, который раз в ~1.5 сек обновляет состояние роботов:
  - онлайн-роботы плавно движутся вдоль waypoint-маршрута (интерполяция позиции),
  - батарея медленно убывает у онлайн-роботов, восстанавливается у RTL,
  - сигнал слегка флуктуирует (±5%),
  - с малой вероятностью робот «теряет связь» (online→offline) и возвращается обратно — для драматизма.
- Состояние поднимается в `index.tsx` и пробрасывается в `MapView`/`RobotPanel`/`DevicesView`, чтобы все вкладки были синхронны.
- Лёгкая CSS-анимация `transition: left/top 1.4s ease` на маркерах роботов — плавное движение по карте без рывков.

## 2. Карта Капшагайского водохранилища (Казахстан)
- Заменить абстрактные «озёра» на стилизованный, но узнаваемый силуэт Капшагайского водохранилища (вытянутое с СВ на ЮЗ, с притоком р. Или).
- Контур водоёма как SVG-path с градиентной заливкой и тонкой береговой линией; добавить:
  - метки опорных точек: «Плотина», «Залив Тас-Откель», «Устье р. Или», «База USV»,
  - компас, масштабную линейку,
  - координатный HUD заменить на реальные координаты Капшагая (≈43.88° N, 77.07° E),
  - подпись «Капшагайское водохранилище · р. Или» в углу.
- Перепозиционировать роботов и пробы так, чтобы они логично лежали внутри контура водоёма.

## 3. След движения (motion traces)
- Хранить в состоянии для каждого онлайн-робота `trail: {x,y}[]` — последние ~25 позиций.
- В `MapView` рисовать SVG `polyline` по trail с градиентом прозрачности (свежий хвост ярче, старый растворяется) в цвет робота.
- Каждому роботу назначить уникальный акцентный цвет (cyan / lime / amber), чтобы следы различались; этот же цвет использовать в карточке устройства и легенде.

## 4. UI/UX «человеческая» отделка
- Хедер: добавить небольшой статус-бар «Капшагай ·온лайн миссии: N · последний пинг: 12с назад», часы по местному времени (Алматы, UTC+5).
- Tabs: переработать в более «живой» сегментный контрол с иконкой + подписью + бейджем счётчика (3 устройства, 2 онлайн, 28 проб).
- Карточки и панели: смягчить тени, добавить тонкие рамки `inset 0 1px` для эффекта стекла, не злоупотреблять glow — оставить только на ключевых элементах (kill switch, активный робот).
- Плавающая панель робота: добавить мини-график батареи за последние 30 сек (sparkline через recharts), читаемые метки «Скорость», «Курс °», текущие GPS координаты, кнопку «Центрировать на карте».
- Sample-кластеры: подписи «N проб» под цифрой, при наведении — тултип с диапазоном дат.
- Микро-анимации: появление панели через `animate-in slide-in-from-right`, fade для модалок, scale на hover у маркеров, pulse только у активного / тревожного.
- Лёгкие «человеческие» детали: разнотолщинные линии в SVG карты, шум/grain через `radial-gradient`, чуть несимметричные отступы в шапке, реалистичные имена миссий (`М-2026-05-17 · Залив Тас-Откель`).
- Toast'ы аварийной остановки и RTL — с явной иконкой и кратким техническим лог-сообщением (`[14:32:08] CMD: ESTOP → AquaBot-01 · ACK`).

## 5. Технические детали
- Новые/изменённые файлы:
  - `src/components/app/useRealtimeSimulation.ts` — хук-симулятор.
  - `src/components/app/MapView.tsx` — новый SVG Капшагая, trails, плавные переходы.
  - `src/components/app/mock-data.ts` — стартовые waypoints, цвета роботов, позиции внутри водоёма.
  - `src/components/app/RobotPanel.tsx` — sparkline, доп. телеметрия, «Центрировать».
  - `src/components/app/types.ts` — поля `color`, `heading`, `speed`, `trail`, `waypoints`, `batteryHistory`.
  - `src/routes/index.tsx` — подключение хука, бейджи в табах, обновлённая шапка.
- Производительность: интервал 1.5 с, trail ограничен 25 точками, никаких тяжёлых ререндеров; графики используются только в аналитике и в одной мини-sparkline.
- Полностью на моках, без бэкенда.

## Что НЕ делается в этой итерации
- Реальная картография (Mapbox/Leaflet) — карта остаётся стилизованной SVG, как и раньше.
- Реальный WebSocket/SSE — только клиентский setInterval-симулятор.
